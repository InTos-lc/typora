## LINUX

### 内存管理工具

### top

`top` 是一个**实时系统监控工具**，而不是用来“测试”CPU或内存的压力工具。它的作用是**动态查看进程占用的 CPU、内存以及系统整体负载**。

如果你希望**施压测试**（让 CPU/内存高占用），应使用 `stress`、`stress-ng` 等工具，然后再用 `top` 观察效果。这里先详细说明 `top` 的用法，最后补充如何配合压力测试。

------

#### 一、top 基本启动与实时界面

在终端直接输入：

bash

```
top
```



界面分为上下两部分：

- **顶部统计区**：系统运行时间、平均负载、任务数、CPU 使用率、内存/交换分区使用情况。
- **进程列表区**：各进程的 PID、用户、优先级、VIRT/RES/SHR 内存、%CPU、%MEM 等。

##### 关键统计行解读（重点）

text

```
%Cpu(s):  2.3 us,  1.0 sy,  0.0 ni, 96.5 id,  0.1 wa,  0.0 hi,  0.1 si,  0.0 st
```



- `us`：用户空间占用 CPU 百分比（应用程序）。
- `sy`：内核空间占用 CPU 百分比。
- `ni`：改变过优先级的进程占用。
- `id`：空闲 CPU 百分比（越高越闲）。
- `wa`：等待 I/O 完成的 CPU 时间（磁盘瓶颈常看这个）。
- `hi` / `si`：硬件中断 / 软件中断。
- `st`：被虚拟化环境偷走的时间。

内存行：

text

```
MiB Mem :  15886.1 total,   1234.5 free,   8765.4 used,   5886.2 buff/cache
MiB Swap:   2048.0 total,   2000.0 free,     48.0 used.   5678.9 avail Mem
```



- `free`：未使用的内存。
- `used`：已用内存。
- `buff/cache`：缓冲区/缓存占用（可被快速回收）。
- `avail Mem`：估算的可用内存（含可回收缓存）。

#### 二、交互式常用快捷键（在 top 界面内直接按）

##### 1. 显示与排序

| 快捷键    | 功能                                   |
| :-------- | :------------------------------------- |
| `1`       | 展开/合并所有 CPU 核心使用率           |
| `M`       | 按**内存使用率**排序（%MEM）           |
| `P`       | 按**CPU 使用率**排序（默认）           |
| `T`       | 按运行时间排序                         |
| `N`       | 按 PID 数字排序                        |
| `R`       | 反向排序（再按一次切换正反）           |
| `f`       | 进入字段管理，选择显示哪些列、排序字段 |
| `F` / `O` | 直接进入排序字段选择                   |

##### 2. 进程操作

| 快捷键    | 功能                                       |
| :-------- | :----------------------------------------- |
| `k`       | 杀死进程（会提示输入 PID 和信号，默认 15） |
| `r`       | 修改进程的 nice 值（优先级）               |
| `u`       | 按**用户**过滤进程（输入用户名）           |
| `V`       | 树状/森林视图显示父子进程关系              |
| `c`       | 切换显示完整的命令行（展示程序参数）       |
| `o` / `O` | 添加过滤条件（例如 `%MEM>10`）             |

##### 3. 显示控制

| 快捷键 | 功能                                  |
| :----- | :------------------------------------ |
| `t`    | 切换 CPU 统计行的显示样式（条状图等） |
| `m`    | 切换内存/交换区分行的显示样式         |
| `h`    | 帮助界面                              |
| `q`    | 退出 top                              |

##### 4. 数值单位切换

- 按 `E`（大写）切换顶部统计区的单位（KiB, MiB, GiB, TiB…）
- 按 `e`（小写）切换进程列表的单位

------

#### 三、命令行参数（启动时预设行为）

bash

```
top -d 2          # 每2秒刷新一次（默认3秒）
top -n 5          # 刷新5次后自动退出（适合脚本取快照）
top -p 1234       # 只监控 PID 为 1234 的进程
top -p 1234,5678  # 监控多个 PID
top -u username   # 只显示指定用户的进程
top -b            # 批处理模式（输出可重定向到文件，避免交互式界面）
top -c            # 启动时就显示完整命令行
top -o %MEM       # 启动时按内存排序
```



**示例：将 top 输出保存到文件**

bash

```
top -b -n 1 > top_snapshot.txt
```

#### 四、如何“测试”CPU/内存占用并观察

`top` 本身不能制造负载，但你可以结合其他工具来做压力测试，然后使用 `top` 观察。

##### 1. 安装压力工具（以 `stress` 为例）

bash

```
# Ubuntu/Debian
sudo apt install stress
# CentOS/RHEL
sudo yum install stress
```



##### 2. 制造负载

bash

```
# 创建2个CPU密集型进程，持续60秒
stress --cpu 2 --timeout 60s

# 创建1个内存分配进程，占用1GB内存，持续60秒
stress --vm 1 --vm-bytes 1G --timeout 60s

# 同时测试CPU和内存
stress --cpu 4 --vm 2 --vm-bytes 512M --timeout 120s
```



##### 3. 在另一个终端运行 top 观察

- 按 `P` 看哪些进程吃满 CPU。
- 按 `M` 看哪些进程占用大量内存（stress 会显示 `stress` 进程）。
- 注意顶部内存的 `used` 和 `avail` 变化。

------

#### 五、补充：只查看内存的命令

如果需要更简单的内存占用快照，可以用：

bash

```
free -h          # 人类可读的内存统计
cat /proc/meminfo
```

查看某个进程的内存详情（RSS、VSZ等）：

bash

```
ps -o pid,user,%mem,%cpu,rss,vsz,comm -p <PID>
```

但实时动态监控，`top` 依然是最方便的选择。
