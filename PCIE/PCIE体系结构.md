# PCIE体系结构

## 第一章：概述

### 1.基本知识

#### 1.1PCI总线的基本知识

**主要目的**：为了连接外部设备。PCI总线，系统总线和处理器体系结构之间依然存在着紧密的联系。

##### 1.1.1PCI总线空间预处理器空间隔离

1.PCI设备具有独立的地址空间，该空间与存储器地址空间通过HOST主桥隔离。处理器需要通过**HOST主桥**才能访问PCI设备，PCI设备需要通过HOST主桥才能访问主存储器。

2.在HOST主桥中含有许多缓冲，使得处理器总线与PCI总线工作在各自的时钟频率中互不干扰。

３.HOST主桥的存在使得PCI设备核处理器可以方便地共享主存储器资源

##### 1.1.2.可扩展性

1.在**PCI 总线**中，**HOST主桥**可以直接推出一条**PCI总线**,该总线可以通过**PCI桥**扩展出一系列**PCI总线**，一个**PCI总线**树上最多挂载256个PCI设备

2.**同一条PCI总线**上的设备可以直接通信，在**同一颗PCI总线树**上的设备也可以直接通信，但需要通过**PCI桥**进行数据转发。

3.**PCI桥**是PCI总线的一个重要组成部分，他的存在时得PCI总线极具扩展性。

4.**PCI 桥**可以管理这个**PCI 总线子树**，在PCI桥的两端分别连接了两条总线，两条总线间的通信需要通过**PCI桥**进行。

**HOST**主桥：连接了CPU/内存世界和PCI总线世界，系统的第一条总线（**BUS0**）由它产生。

**总线层次**：

对**PCI桥1**来说，Bus0为**上游总线**，Bus1为**下游总线**

对**PCI桥2**来说， BUS1为**上游总线**， Bus2为**下游总线**

通信路径：CPU -> HOST主桥 -> Bus 0 -> PCI桥1 -> Bus 1 -> PCI桥2 -> Bus 2 -> 设备D

每一个桥设备都会检查地址，决定是否将数据包转发到自己的下游总线。
