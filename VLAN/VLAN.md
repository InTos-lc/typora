# VLAN

## vlan定义

VLAN是通过软件配置，将一个物理的局域网在逻辑上划分成多个虚拟的广播域的技术。每个VLAN就像一个独立的，虚拟的交换机在运行。

## vlan优点

1.  提高安全性：
   - 不同VLAN之间的设备在二层（数据链路层）是隔离的。无法直接通信，
   - 必须通过三层设备进行
2. 控制广播风暴：
   - 广播流量被限制在产生它的VLAN内
3. 简化网络管理和项目管理
   - 逻辑分组，不受物理位置的限制

## vlan是如何工作的？----vlan标签（802.1Q）

交换机如何知道一个数据帧属于哪一个vlan呢？

是IEEE 802.1Q协议，它定义了VLAN的封装标准。

- **普通以太网帧** ：目的MAC|源MAC|类型/长度|数据|帧校验序列（FCS）

- **带VLAN标签的帧（802.1Q帧）**: 
  - 目的MAC|源MAC|0x8100（**TPID**）|标签信息（TCI）|类型/长度|数据|FCS

这个**标签信息（TCI）**里最重要的就是**VLAN ID（12位）**

- VLANIP的范围是1-4094.

- **VLAN 1**通常是交换机的默认本征VLAN

**打标签和去标签的过程：**

1. **Access端口接收帧（链接PC）：**PC发送的是普通帧（无标签）。交换机会根据端口配置的VLAN ID，给这个帧 **打上对应的VLAN标签**。
2. **Access端口发送帧（链接PC）**：交换机在发送前，**会去掉VLAN标签，还原成普通帧**，因为PC通常不认识VLAN标签。
3. **Trunk端口发送帧（链接交换机）：**帧在Trunk链路上传输时，**会保留VLAN标签**（除了本征）
4. **Trunk端口接收帧（链接交换机）：**交换机读取帧中的VLAN标签，就知道该将它转发到那个VLAN了

## VLAN的端口类型

交换机端口有三种常见的工作模式，决定了它如何处理VLAN标签

| 端口类型   | 用途                                                         | 处理VLAN标签的方式                                           | 可属于VLAN数量 |
| ---------- | ------------------------------------------------------------ | ------------------------------------------------------------ | -------------- |
| Access端口 | 连接**不支持VLAN**的终端设备（PC， 打印机，服务器普通网口）  | **接收：**给所有帧打上该端口配置的PVID（端口VLAN ID）<br />**发送：**仅当帧的VLANID与PVID相同时，才去掉标签发出 | 1个            |
| Trunk端口  | 连接**支持VLAN**的网络设备（交换机， 路由器， 服务器Trunk网口） | **接收：**接受带标签的帧。如果收到无标签帧，则打上，**本征VLAN（Native  VALN）的标签**<br />**发送：**帧的VLAN ID与本征VLAN相同时，**去掉标签**发出；其他VLAN的帧**带标签发出** | 多个           |
| Hybrid端口 | （常见于华为设备）兼具Access和Trunk的灵活性                  | **接收：**与Trunk端口类似<br />**发送：**可针对每个VLAN配置是**带标签**发送还是**不带标签发送**。 | 多个           |
